## 二、架构关键变更

### 2.1 Compose Compiler：从 Library 到 Kotlin Plugin（最大变更）

**原方案（过时）**：
```kotlin
composeOptions {
    kotlinCompilerExtensionVersion = "1.5.15"
}
```

**修正方案（Kotlin 2.0+）**：
```kotlin
// 在 Version Catalog 中定义 plugin
[plugins]
compose-compiler = { id = "org.jetbrains.kotlin.plugin.compose", version.ref = "kotlin" }

// 在 app/build.gradle.kts 中引用
plugins {
    alias(libs.plugins.compose.compiler)  // ← 替代 composeOptions {}
}

// 删除以下旧代码：
// composeOptions { kotlinCompilerExtensionVersion = "..." }
```

> **官方说明**：从 Kotlin 2.0 开始，Compose Compiler 不再作为独立库发布，而是作为 Kotlin 编译器插件，版本与 Kotlin 完全同步。

### 2.2 Hilt ViewModel 在 Compose Navigation 中的正确用法

**原方案**（不精确）：
```kotlin
@Composable
fun MyScreen(viewModel: MyViewModel = hiltViewModel()) {
```

**修正方案**：
```kotlin
// 在 NavHost composable 中使用 hiltViewModel()
@Composable
fun HeliumNavHost(navController: NavHostController) {
    NavHost(navController, startDestination = "records") {
        composable<Routes.RecordList> {  // ← 类型安全路由
            val viewModel: RecordListViewModel = hiltViewModel()
            RecordListScreen(viewModel = viewModel)
        }
    }
}
```

### 2.3 新增：平板适配方案

原方案仅在兼容性测试中提及平板，但未给出具体适配方案。

**新增 Material3 Adaptive 实现**：
```kotlin
// 使用 adaptive layout 自动适配平板
import androidx.compose.material3.adaptive.layout.ListDetailPaneScaffold

@Composable
fun RecordListDetailScreen() {
    ListDetailPaneScaffold(
        // 平板：左侧列表 + 右侧详情（双窗格）
        // 手机：全屏列表 → 点击跳转详情（单窗格）
        listPane = { RecordListPane(...) },
        detailPane = { RecordDetailPane(...) }
    )
}
```

## 三、数据库版本迁移策略

### 3.1 Room 2.8.x 新特性

Room 2.8.x 增强了 KSP 支持、改进了 Flow 查询性能、增加了自动迁移能力：

```kotlin
// Room 2.8+ 的 AutoMigration 能力
@Database(
    entities = [DetectionRecordEntity::class, EquipmentEntity::class, ...],
    version = 2,
    autoMigrations = [
        AutoMigration(from = 1, to = 2)
    ]
)
abstract class HeliumDatabase : RoomDatabase() { ... }
```

### 3.2 预留升级路径

| 数据库版本 | 变更内容 |
|-----------|---------|
| v1 (MVP) | detection_records + equipment + reminders |
| v2 (V1.1) | 新增字段：export_history, user_tags |
| v3 (V2.0) | 新增表：user_accounts, cloud_sync_metadata |

## 五、安全性补充建议

原方案未覆盖以下安全要点：

| 安全项 | 建议方案 | 优先级 |
|--------|---------|--------|
| 数据加密 | Room + SQLCipher 加密本地数据库 | P2 |
| APK 防篡改 | ProGuard/R8 混淆 + 签名校验 | P1 |
| 导出文件安全 | 导出 JSON/CSV 前清理临时文件 | P1 |
| 敏感信息 | 检测参数不涉及个人信息，无隐私合规风险 | — |

## 六、审查结论

| 类别 | 原方案评估 | 修正后状态 |
|------|-----------|-----------|
| 技术选型 | 方向正确 | ✅ 保持 MVVM + Compose + Room + Hilt |
| 版本号 | **4 处过期** | ✅ 全部更新至最新稳定版 |
| Compose 编译器 | **架构性错误** | ✅ 修正为 Kotlin Compose Plugin |
| 缺失依赖 | **3 处缺失** | ✅ 补充 activity-compose、lifecycle、hilt-navigation-compose |
| 平板适配 | 方案欠缺 | ✅ 补充 Material3 Adaptive |
| 数据库迁移 | 方案欠缺 | ✅ 补充 Room AutoMigration 策略 |
| 安全方案 | 方案欠缺 | ✅ 补充混淆+加密建议 |

> **总体评价**：策划方案的技术方向和架构设计正确，但版本号需要更新至 2026 年最新稳定版。最关键的修正是 **Compose Compiler 从独立 library 变为 Kotlin 编译器插件**，这是 Kotlin 2.0+ 与旧版本的根本性差异。
